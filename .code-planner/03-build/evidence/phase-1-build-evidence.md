## Existing code discovery

Discovery confirmed during WU-008 itself:

- **Project layout**: `src/llm_wiki/{cli,config,db,jobs,llm,pipeline,schema}`,
  `pyproject.toml` declares `wiki = "llm_wiki.cli:main"` console script and
  Python `>=3.10`; `requires-python` is honored by the validation environment
  (Python 3.13.5).
- **Entry point**: `python3 -m llm_wiki.cli --help` works without
  installation because the package lives under `src/`. PYTHONPATH=src is the
  standard invocation in README and validation commands.
- **CLI**: argparse-based with subparsers for every Phase 1 command. Exit
  codes match the spec: success `0`, user input error `2`, dependency
  failure `3` when an external dependency is truly unavailable.
- **Settings**: YAML via `pyyaml` 6.0.2 with a small fallback parser so the
  CLI still runs in environments without PyYAML. `mask_sensitive()` in
  `common.py` redacts any key matching `secret|token|password|api_key|apikey|key`.
- **DB schema**: `sources`, `source_chunks`, `embeddings`, `jobs`,
  `agent_runs`, `artifacts`, `review_candidates`, `human_decisions`,
  `retry_instructions`, `prompt_versions`, plus `source_chunks_fts` virtual
  table. Matches the SQLite schema draft with one FTS virtual table.
- **Embedding**: `pipeline/embed.py` calls
  `fastembed.TextEmbedding(model_name=...)`. It now includes a configurable
  operation timeout and falls back to deterministic `fallback-hash-v1` only if
  fastembed import/model loading/inference fails. The risk-resolution E2E used
  `backend: fastembed` successfully.
- **Candidate contract**: `schema/candidates.py` enforces the required
  `task_type/source_id/schema_version` envelope plus the five empty-array
  buckets. `human_decision`, `retry_instruction`, `approved`, `rejected`,
  `replaced` are rejected in LLM output.
- **Env handling**: `.env` is **only** read via `common.read_env_file()` to
  populate `wiki doctor` reports; secrets are masked in output and never
  included in artifacts. The real `.env` file in the repo root was
  loaded into the test subprocesses' environment only (via `set -a;
  source .env; set +a`) so the configured `wiki models test` and
  embed steps could exercise the real endpoint; the secret value
  itself was never echoed, never written into an artifact file,
  never stored in `error_json`, and never copied into the repo's
  `.env.sample` or workspace tree. Only booleans (`api_key_present`)
  and the redacted `LL***EY` fragment appear in captured payloads.

No duplicate-risk issues found. The `cli/__init__.py` aggregate is the only
top-level dispatch file; other commands live in their own module.

   `wiki models test` configured-failure branches no longer persist
   bearer tokens or any header value matching `Bearer `. The

   bounded-operation context manager covers POSIX and non-POSIX
   hosts. `fastembed_timeout_seconds` defaults to 30s and a stalled
   embedder now falls back within the budget.

   `.env` E2E ended with zero jobs in `running`/`queued`/`failed`.
