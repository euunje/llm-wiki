## Fixed items

### M-1 — Embedder output validation

- Added a new helper `_validate_embedder_output(vectors, expected_count)`
  in `src/llm_wiki/pipeline/embed.py`. It rejects:
  - non-list output
  - list length mismatch (short output / missing vectors)
  - empty vector list
  - zero-dimension vectors
  - non-iterable or non-numeric vector members
  - dimension mismatch across vectors
  - all-zero vectors
  It returns `(None, reason)` on failure or `(validated, None)` on
  success. The reason is included verbatim in the embedder output
  failure path so the artifact JSON is self-describing.
- Wired the validator into `embed_target`. When the validator rejects
  the embedder output, the function now sets `embedder = None` and
  `embedder_error = "embedder output invalid: <reason>"`, so the
  subsequent fallback path runs deterministically and the
  `backend_detail.reason` carries the explicit validation reason.
- The previous silent `zip(targets, vectors)` truncation is no longer
  reachable; the validator runs **before** `zip` is used to build
  embedding rows.
- Behavior contract: malformed embedder output always falls back to
  `fallback-hash-v1` with `backend: "fallback_hash"` and an explicit
  `backend_detail.reason`. Success (`backend: "fastembed"`) is
  reachable only when the validator passes.
