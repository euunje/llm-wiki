# LLM-Wiki 업그레이드 계획 (to_upgrade_plan)

본 문서는 기존 LLM-Wiki 아키텍처의 확장성, 유연성 및 관리 편의성을 높이기 위한 고도화 방향을 정의합니다.

## 1. Docker Compose 운영 환경 구축
*   **목적**: 배포 및 실행 환경의 일관성 확보
*   **주요 작업**:
    *   `docker-compose.yml` 구성
    *   **볼륨(Volume) 마운트**: Obsidian Vault 데이터 및 DB/로깅 데이터의 영구 보존(Persistence) 보장
    *   **환경 변수(.env) 통합**: 대시보드에서 관리하는 `.env` 설정을 Docker 컨테이너와 일원화하여 참조

## 2. 분류 체계 유연화: `non_categories` 버퍼 도입
*   **목적**: 정의되지 않은 새로운 개념(Entity)이나 LLM의 분류 오류(Hallucination)에 유연하게 대응
*   **주요 작업**:
    *   `non_categories` 폴더 신설: 조건에 맞지 않거나 확신도(confidence)가 낮은 문서를 일차적으로 수용하는 버퍼(Inbox) 역할
    *   **안전한 재링크(Rebuild Link) 로직 구현**: 문서가 `non_categories`에서 정식 카테고리 폴더로 이동(move)할 때, 해당 문서를 참조하는 모든 마크다운 파일의 내부 링크를 자동으로 찾아 업데이트
    *   **상태 메타데이터 추가**: 마크다운 파일 Frontmatter에 `status`, `confidence`, `processed_at`, `source_file` 등의 속성 추가

## 3. 대시보드 기반 시스템 운영 및 관리 강화
*   **목적**: 개발자의 코드 수정 없이 사용자가 직접 파이프라인을 제어하고 품질을 튜닝
*   **주요 작업**:
    *   **LLM 설정 관리**: 웹 서버에서 직접 `.env` (API Key, 모델 등) 수정
    *   **프롬프트 버전 관리 체계 구축**:
        *   **Draft 모드**: 자유롭게 수정하고 테스트해볼 수 있는 임시 상태
        *   **Publish (운영 반영)**: 테스트 완료 후 운영 파이프라인에 적용하는 시점에만 버전(예: `v1.0`, `v1.1`) 스냅샷을 생성. 문제 발생 시 이전 확정 버전으로 롤백 지원
    *   **로깅 및 재처리(Reprocess)**: 파싱 결과 로그를 확인하고, 분류 실패한 문서에 대해 개선된 프롬프트로 '다시 처리하기' 지원
    *   **분류 대기열(Inbox) UI**: `non_categories` 항목들을 대시보드에서 빠르게 검토하고 적절한 카테고리를 할당하여 이동/반영시키는 기능
    *   **MCP Protocol 설정**: 생성된 지식 베이스를 외부 LLM 도구(Claude Desktop 등)에서 접근하여 활용할 수 있도록 연동 설정 지원

## 4. 다양한 문서 포맷(PPT, Word, PDF 등) 지원 아키텍처
*   **목적**: 텍스트 외의 복잡한 비정형 문서에서 지식을 추출하고 원본을 안전하게 관리
*   **주요 작업**:
    *   **파서(Parser) 모듈화**: `DocumentParser` 공통 인터페이스를 설계하여 `MarkdownParser`, `PDFParser`, `PPTXParser` 등을 플러그인처럼 교체/확장할 수 있는 구조 적용
    *   **Raw 문서(원본) 분리 보관 (Data Lake 전략)**:
        *   무거운 원본 파일(PPT, DOCX 등)은 Obsidian Vault에 직접 넣지 않고, 파이프라인 서버 측의 별도 전용 저장소(예: `/data/raw_docs/`)에 보관하여 Vault의 경량성 유지
        *   생성된 마크다운 위키 문서 본문이나 메타데이터에 원본 파일을 다운로드/열람할 수 있는 대시보드 파일 서버 링크(URL) 제공
        *   단, 추출된 핵심 이미지/차트는 Vault 내부의 `assets` 폴더에 저장하여 본문에 즉시 렌더링되도록 구성
    *   **대시보드 내 청킹(Chunking) 전략 세부 설정 기능**:
        *   문서 포맷에 따른 최적의 텍스트 분할 방식 선택 지원 (예: PDF/PPT는 슬라이드/페이지 단위 분할, Word는 문단/길이 기반 분할)
        *   `Chunk Size`(분할 단위 크기) 및 `Overlap`(문맥 겹침 정도) 값을 UI에서 유연하게 설정할 수 있도록 제공하여 정보 추출 해상도 제어
