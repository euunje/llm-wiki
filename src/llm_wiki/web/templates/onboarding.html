Mapping flow
  1. 기존 vault root를 선택합니다.
  2. 위 폴더 브라우저에서 Inbox/Raws/Wiki 폴더로 이동합니다.
  3. 상단 또는 각 폴더 행의 Inbox/Raws/Wiki 버튼으로 바로 매핑합니다.
  4. Wiki 내부 세부 폴더명은 Advanced에서 입력하고 저장합니다.

{# Step 5: Pipeline overview #}

Inbox → 자동 처리 → Mapping/Wiki

Pipeline은 Inbox 입력이 검색 가능한 정규화 청크, embedding 인덱스, LLM 추출/요약 후보를 거쳐 Mapping 검토와 최종 Wiki 출력으로 자동 이동하는 처리 흐름입니다.

Inbox input
00_Inbox/memo
00_Inbox/files
00_Inbox/text
→
Automatic pipeline
ingest → normalize → chunk → embed → extract
→
Mapping / Wiki
mapping candidates → review → confirm
{# Pipeline step explanations #}

Pipeline 단계 설명

즉, 원본은 먼저 안전하게 수집되고, 검색 가능한 형태로 정리되며, LLM이 후보를 만들고, 사용자가 Mapping에서 확정하면 최종 Wiki로 저장됩니다.

ingest 원본 파일을 수집하여 raw 영역에 저장합니다. 메타데이터(파일명, 경로, 타입)를 기록합니다.
normalize 다양한 포맷의 내용을 정제하고 표준 형식(마크다운/텍스트)으로 변환합니다.
chunk 내용을 의미 있는 단위(문단/섹션)로 분할하여 검색 및 임베딩에 적합하게 만듭니다.
embed 각 청크를 벡터로 변환하여 유사도 검색(embedding/vector search)에 사용합니다.
extract_claims LLM으로 원문에서 사실/주장/근거를 추출하여 후보를 생성합니다.
summarize 추출된 정보를 요약하여 wiki 페이지 초안을 작성합니다.
map 기존 wiki 개념과 새 후보를 매칭하여 병합/생성/보류 결정을 준비합니다.
link 개념 간 관계(관련/상위/하위/근거)를 연결하여 그래프를 구성합니다.
compile / wiki 승인된 후보를 wiki 페이지로 확정하고 마크다운 파일로 저장합니다.
