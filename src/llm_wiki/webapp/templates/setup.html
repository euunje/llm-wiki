LLM-Wiki 시작하기

문서를 수집하고 지식 베이스를 자동 정리할 수 있도록 프로젝트 폴더와 LLM 연결 정보를 설정합니다.

프로젝트 루트 또는 Obsidian Vault 루트 폴더를 선택하세요.

🆕 새 Vault로 시작

새로 시작합니다. 표준 폴더(`Inbox/`, `raw/`, `wiki/`, `schema/`)를 자동으로 만들어 바로 사용할 수 있게 준비합니다.

🔄 기존 Vault 연결

이미 쓰고 있는 Obsidian Vault를 연결합니다. 기존 폴더 구조를 유지하면서 역할만 매핑합니다.

생성될 기본 폴더 구조

새 Vault 시작을 선택하면 아래 구조를 기준으로 생성합니다.

📂 [Vault Root]/
├── 📁 Inbox/                       ← 사용자 입력 파일 드롭 영역
│   ├── 📁 Files/                   ← 바이너리/첨부 파일
│   ├── 📁 Markdown/                ← 마크다운 문서
│   ├── 📁 Text/                    ← 텍스트 문서
│   ├── 📁 _Failed/                 ← 처리 실패 원본
│   └── 📁 _Review/                 ← 리뷰 대기 원본
├── 📁 raw/                         ← 처리 완료 원본 아카이브
├── 📁 wiki/
│   ├── 📁 sources/                 ← 소스 요약 페이지
│   ├── 📁 entities/                ← 사람, 조직, 모델, 제품
│   ├── 📁 concepts/                ← 지속적 개념
│   ├── 📁 synthesis/               ← 저장된 쿼리/종합 페이지
│   ├── 📁 non_categories/          ← 후보 리뷰 큐
│   ├── 📁 assets/                  ← 추출된 자산
│   ├── 📄 index.md                 ← 생성된 카탈로그
│   ├── 📄 log.md                   ← 생성된 운영 로그
│   └── 📁 .obsidian/               ← 선택적 Obsidian 기본 설정
├── 📁 schema/AGENTS.md             ← 라우팅 규칙 + 스키마
└── 📁 .wiki/                       ← 내부 설정 + 상태 DB

폴더 역할 매핑

📁 감지된 폴더
프로젝트 경로를 입력하면 폴더를 스캔합니다.

💡 배지를 드래그하거나 옆의 아이콘을 사용하세요: 📥 Inbox, 📦 Raw 아카이브, 📂 Wiki, ⚙️ Schema에 매핑합니다.

📥 Inbox 폴더 (사용자 입력)

원본 파일이 들어오는 입력 지점입니다. 파이프라인은 이 폴더에서 파일을 읽습니다.

선택한 Inbox 아래에 Files, Markdown, Text, _Failed, _Review가 자동 생성됩니다. 실패 항목은 _Failed로 이동되고, 재시도하면 다시 처리 대기 상태로 돌아갑니다.

📦 Raw 아카이브 폴더 (처리 완료 원본)

처리가 완료된 원본 파일이 보관되는 아카이브 영역입니다.

📂 Wiki 폴더 (AI 노트 출력)

AI가 생성한 노트(엔티티, 개념, 종합)와 리뷰 항목이 저장됩니다. 루트에 사용하려면 .을 입력하세요.

⚙️ Schema 폴더 (라우팅 규칙)

라우팅 규칙, 템플릿, 지시사항(AGENTS.md)이 저장되는 설정 영역입니다.

🧭 전체 폴더 맵 (사용자에게 표시되고 설정에 저장됩니다)
