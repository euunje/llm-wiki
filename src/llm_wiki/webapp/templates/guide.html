📂 1. Core Directory Roles

LLM-Wiki divides files into 3 logical areas. By configuring these correctly, you can integrate the pipeline directly into any existing Obsidian Vault.

📥 Input

Inbox (Raw 소스)

Where you drop your raw files (PDFs, PPTX, Docx, HTML, TXT). The ingestion pipeline monitors this directory and parses documents from here.

Default: `raw/`
📂 Output

Wiki (Obsidian Notes)

Where the AI-generated categorized notes are stored. This folder is scanned by Obsidian to display your interactive knowledge graphs.

Default: `wiki/`
⚙️ Rules

Schema (Taxonomy)

Houses your system rules (`AGENTS.md`). This file defines routing guidelines and tells the LLM how to classify candidates as entities, concepts, or review items.

Default: `schema/`

🏷️ 2. Auto-Generated Categories

Within the configured **Wiki Folder**, the pipeline writes markdown notes into specific subdirectories to organize files logically:

📄 sources/

Holds document summary notes. Each note contains takeaways, frontmatter metadata, and clickable file:/// links to open local raw files.

👤 entities/

Individual notes representing real-world nouns like **People, Models, Organizations, Products** (e.g. `karpathy.md`, `gpt-4.md`).

💡 concepts/

Notes representing core abstract ideas, techniques, or subject domains (e.g. `RAG.md`, `Fine-tuning.md`).

📊 synthesis/

Overview or summary pages that combine key concepts and takeaways extracted from multiple sources (cross-document reasoning).

📥 non_categories/ (Inbox Queue)

A review queue for ambiguous or externally owned candidates. Guide-like content is held for `8000-web-config`; map/MOC-like content is held for `mcp-map`.

🖼️ assets/

Extracts visual materials (slide layouts, shapes, diagrams) from PDFs and PowerPoint slides, allowing inline image links inside Obsidian.

⚡ 3. LLM Ingestion Pipeline Flow

When you run the ingestion process, the pipeline carries out five steps asynchronously:

1

Parse Document

Extracts raw text chunks from documents based on chosen strategy (Sliding Window or Page-based). Visual charts are extracted to the `assets/` folder.

2

LLM Extraction

The LLM reviews the document contents, generates takeaways, and extracts candidates classified as entity, concept, or review using routing rules in `AGENTS.md`.

3

Confidence Filter & Staging

Entity and concept candidates are staged to their configured folders. Review candidates, including guide-like or map-like items, go to `non_categories/` (Inbox) instead of becoming normal wiki pages.

4

Deduplication & Relinking

The pipeline compares extracted slugs to existing wiki documents. It merges references and writes standard `[[wikilinks]]` in Obsidian format.

5

Index Rebuild
